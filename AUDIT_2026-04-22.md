## ­čö┤ Critical issues found (and fixed)

### 1. Service worker was broken on install
**File:** `public/sw.js`

`cache.addAll([...])` is atomic: a single 404 rejects the whole install and
the service worker never activates. `ASSETS_TO_CACHE` listed four files that
didn't exist on disk (all zero-byte stubs):

- `/assets/css/main.css` ÔÇö empty file
- `/assets/css/mobile.css` ÔÇö empty file
- `/js/offline.js` ÔÇö empty file
- (Plus `/views/mobile/index.html` which did exist ÔÇö kept)

**Fix:** Deleted the empty files. Rebuilt `ASSETS_TO_CACHE` with only files
that actually exist. Bumped `CACHE_NAME` v1 Ôćĺ v2 to force clients to
re-install.

### 2. Manifest icons pointed at nothing
**File:** `public/manifest.json`

Referenced `/assets/img/icons/icon-192.png` and `-512.png`. The
`public/assets/img/` directory **doesn't exist**. Chrome refuses to install
the PWA when manifest icons 404.

**Fix:** Replaced `icons` array with `[]` until real icons are added. PWA
install still available via the browser UI; just no custom icon yet.
TODO added to README.

### 3. Mobile view redirected to a broken orphan login
**File:** `public/views/mobile/index.html`

Logged-out users were redirected to `../office/index.html`, which was:
1. A duplicate, older-looking login page (light theme, email/password form).
2. Calling `AuthService.login(email, pass)` ÔÇö **a method that doesn't
   exist** in `auth.js` (which only exports `loginWithGoogle` and `logout`).

So: any field tech who opened the mobile app logged out Ôćĺ redirected to a
form that couldn't actually sign them in.

**Fix:** Deleted `public/views/office/index.html`. Changed mobile redirect
to `/index.html` (the real Google sign-in page).

### 4. `_redirects` in repo root was silently ignored
**File:** `_redirects`

Netlify reads `_redirects` from the **publish directory** (`public/`), not
the repo root. The root-level file was dead code. Pretty-URL aliases
(`/dashboard`, `/dispatch`, `/mobile`) and the SPA fallback only worked
because `netlify.toml` duplicated the SPA rule ÔÇö the aliases never worked.

**Fix:** Moved to `public/_redirects`. Removed duplicate SPA rule from
`netlify.toml`.

### 5. Duplicate utility functions in two forms
**Files:** `public/views/forms/invoice.html`, `sales_call.html`

Both files redefined `setVal`, `autoGrow`, or `setChk` inline instead of
importing from `/js/form-utils.js`. Meaning: bug fixes to `form-utils.js`
silently didn't apply to these two views, and the input-highlighting logic
was copy-pasted.

**Fix:** Both now import from `form-utils.js`. `window.autoGrow = autoGrow`
keeps the global binding for inline `oninput="autoGrow(this)"` handlers.

## ­ččí Non-critical cleanup

### 6. Empty files throughout
Deleted (all zero-byte, no real references):
- `public/js/app.js`
- `public/js/offline.js`
- `public/assets/css/main.css`
- `public/assets/css/mobile.css`
- `public/assets/css/paper.css`

### 7. Relative import paths
Every JS module import is now **absolute** (`/js/...`). Previously a mix:
- `db.js` used `./firebase-config.js`
- `form-utils.js` used `./db.js`
- `dispatch.js` used `../firebase-config.js`
- Every view in `views/forms/` used `../../js/...`

Mixing depths is brittle; moving a view to a different folder would have
silently broken its imports.

### 8. Extra security headers
Added to `netlify.toml` (cheap, no behavior change):
- `X-Frame-Options: DENY`
- `X-Content-Type-Options: nosniff`
- `Referrer-Policy: strict-origin-when-cross-origin`

### 9. Missing repo docs
Added `README.md` (run/deploy/structure/collections/smoke-tests) and
`.env.example` (for when functions grow real secrets).

- **Tests** ÔÇö zero automated tests today. At minimum, a Playwright smoke
  test that logs in and loads each view would catch deploy regressions.
- **CI** ÔÇö no GitHub Actions; Netlify does the build and that's it.
